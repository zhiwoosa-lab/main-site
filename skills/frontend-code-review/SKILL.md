# Frontend Code Review

## 用途

审查前端实现是否真正服务这个网站的审美和阅读体验，而不是只看“能不能跑”。

## 何时使用

- 页面、样式、组件、动画、媒体展示发生变化后
- 上线前做 UI 走查
- 移动端阅读、布局或导航出现疑问时
- 需要确认视觉是否仍保持统一人格时

## 行为规则

- 先检查是否符合整体视觉语言，再检查局部实现。
- 必查项目：
  - 字体层级和阅读节奏是否一致
  - 正文宽度、行高、段落间距是否适合长文阅读
  - 移动端是否需要过多缩放、横向滚动或误触
  - 卡片、边框、标签、按钮是否复用一致
  - 动画是否过度、是否影响阅读或性能
  - 图片、视频、地图在不同视口下是否稳定
  - 导航是否让人知道自己位于哪个主题或路径中
  - 空状态、长标题、超长标签、中文英文混排是否正常
- 对无障碍至少检查：
  - 对比度
  - 焦点态
  - alt 文本
  - 键盘访问
  - 语义结构
- 不只指出缺陷，也要指出它会如何破坏个人网站气质或阅读体验。

## 输出要求

- 先给 findings，按严重程度排序。
- 每条 finding 说明：
  - 问题
  - 影响
  - 位置
  - 建议修复方式
- 最后再补充剩余风险或测试空白。

## 注意事项

- 不要把评论局限在“像素级一致”，这里更重要的是整体节奏和人格是否被破坏。
- 避免用复杂动效掩盖内容不足。
- 移动端不是桌面端的缩小版，要单独看待阅读、导航和媒体体验。

## 适合本项目的调用场景

- “请用 `frontend-code-review` 审查这次首页改版。”
- “文章页刚重做完，检查移动端长文阅读体验。”
- “新增地图与媒体模块后，做一次前端专项 review。”
