把监控从本地搬到线上

2026.05.30 · 约 3 分钟

遇到了什么问题

我设了一个每 30 分钟检查 Apple 翻新机库存的定时任务。一开始挺好，但后来发现我电脑一合盖、一休眠，任务就停了。也就是说，我在睡觉、出门、或者只是合上电脑的时候，这个监控形同虚设。

更讽刺的是，稀缺的 Mac mini 和 Mac Studio 翻新机可能就在那个时间段上架，然后被抢光。

怎么解决的

方案很简单——把任务从本地 cron 迁出去。选的是 GitHub Actions，理由：

  • 免费，不需要额外服务器
  • 24 小时在线，电脑关不关都无所谓
  • 我的脚本只是 curl + grep，GitHub 的 runner 足够胜任

迁移过程遇到了一些小坑：GitHub 的工作流文件缓存问题、脚本输出格式兼容问题。但整体半个小时左右搞定了。

学到的

依赖本地设备的自动化，本质上不是自动化。 只要设备可能关机、休眠、断网，这个定时任务就是不可靠的。如果任务确实需要持续运行，就应该放在持续运行的地方。

另外一点：以前用 OpenClaw cron 每跑一次都要调一次 AI 模型，消耗 token 和费用。迁移到 GitHub Actions 后，纯 shell 脚本执行，零 AI 调用成本——用最合适的工具做最合适的事。

下一步

这次的思路可以复用：其他需要持续运行的监控类任务，也可以优先考虑 GitHub Actions 而不是本地定时器。
